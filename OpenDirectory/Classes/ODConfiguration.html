Discussion

@method addTrustType:trustAccount:trustPassword:username:password:joinExisting:error:

@abstract Adds a trust account with the provided name and password using the credentials provided by the user.

@discussion Adds a trust account with the provided name and password using the credentials provided by the user. User can request that the trust be forcibly created (replacing existing trust if found in directory). A trust should be established only after enough configuration is available and the configuration been saved. If the trust is required, then the configuration can be deleted if necessary upon failure.
