Discussion

@typedef os_block_t

@abstract Generic type for a block taking no arguments and returning no value.

@discussion When not building with Objective-C ARC, a block object allocated on or copied to the heap must be released with a -[release] message or the Block_release() function.

The declaration of a block literal allocates storage on the stack. Therefore, this is an invalid construct: os_block_t block; if (x) { block = ^{ printf(“true\n”); }; } else { block = ^{ printf(“false\n”); }; } block(); // unsafe!!!

What is happening behind the scenes: if (x) { struct Block tmp_1 = …; // setup details block = &tmp_1; } else { struct Block tmp_2 = …; // setup details block = &tmp_2; }

As the example demonstrates, the address of a stack variable is escaping the scope in which it is allocated. That is a classic C bug.

Instead, the block literal must be copied to the heap with the Block_copy() function or by sending it a -[copy] message.
