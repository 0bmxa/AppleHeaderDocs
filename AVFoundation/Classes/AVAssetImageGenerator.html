+ assetImageGeneratorWithAsset:

Returns an instance of AVAssetImageGenerator for use with the specified asset. @param asset The asset from which images will be extracted.

+ (instancetype)assetImageGeneratorWithAsset:(AVAsset *)asset

Return Value

An instance of AVAssetImageGenerator

Discussion

@method         assetImageGeneratorWithAsset:

This method may succeed even if the asset possesses no visual tracks at the time of initialization. Clients may wish to test whether an asset has any tracks with the visual characteristic via [AVAsset tracksWithMediaCharacteristic:].

                Note also that assets that belong to a mutable subclass of AVAsset, AVMutableComposition or AVMutableMovie,
                may gain visual tracks after initialization of an associated AVAssetImageGenerator.

                However, the results of image generation are undefined if mutations of the asset occur while images
                are being generated. 

                AVAssetImageGenerator will use the default enabled video track(s) to generate images.

– generateCGImagesAsynchronouslyForTimes:completionHandler:

Returns a series of CGImageRefs for an asset at or near the specified times. @param requestedTimes An NSArray of NSValues, each containing a CMTime, specifying the asset times at which an image is requested. @param handler A block that will be called when an image request is complete.

- (void)generateCGImagesAsynchronouslyForTimes:(NSArray<NSValue*> *)requestedTimes completionHandler:(AVAssetImageGeneratorCompletionHandler)handler

Discussion

@method         generateCGImagesAsynchronouslyForTimes:completionHandler:

Employs an efficient “batch mode” for getting images in time order. The client will receive exactly one handler callback for each requested time in requestedTimes. Changes to generator properties (snap behavior, maximum size, etc…) will not affect outstanding asynchronous image generation requests. The generated image is not retained. Clients should retain the image if they wish it to persist after the completion handler returns.
