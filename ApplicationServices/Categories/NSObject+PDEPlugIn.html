Overview

@protocol PDEPlugIn

– initWithBundle:

Called once when the PDE plug-in is loaded. When using a sandbox application, if the PDE plug-in does not declare itself sandbox-compatible, the PDE is loaded to get its name and is then unloaded. The PDE is reloaded when it is selected in the PDE menu or when the user selects the print button.

- (BOOL)initWithBundle:(NSBundle *)theBundle

Parameters

theBundle

The plug-in’s bundle.

Return Value

Returns YES if initialization succeeded and NO otherwise.

– PDEPanelsForType:withHostInfo:

Returns an array of instances conforming to the PDEPanel informal protocol.

- (nullable NSArray *)PDEPanelsForType:(NSString *)pdeType withHostInfo:(id)host

Parameters

pdeType

The type of print panels that should be returned. These kinds are defined in PMPrintingDialogExtensions.h and include: kGeneralPageSetupDialogTypeIDStr - Panels that should be shown in the page setup dialog. kGeneralPrintDialogTypeIDStr - Panels that should be shown in the print dialog.
host

A print system provided instance that implements methods from the informal protocol PDEPlugInCallbackProtocol. The PDEPlugIn can use this parameter to obtain the current printer or print setting in order to filter the available panels if there are printer specific PDEPanels in the PDEPlugIn.

Return Value

An array of instances implementing methods from the PDEPanel informal protocol. Each instance represents a panel that will be shown in the printer dialog. The method may return either nil or an empty array if no panels should be used.
